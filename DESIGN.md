# 幸存者法庭 · 设计文档

> 把一个被所有人忽视的统计学真相，做成一个让所有人亲身经历顿悟的游戏。

---

## 一、项目定位

**赛题**：学习异化工厂（题目四）  
**核心创新**：叙事在前，知识在后。玩家不是「先读知识再答题」，而是「先遭遇问题，在错误中被迫理解」。  
**路演一句话**：「我正在学计量经济学，教材里有一节讲选择偏误，一段文字我看完觉得懂了——但其实没懂。我想把真正让人「懂了」的那个瞬间，做成任何人都能经历的游戏。」

---

## 二、故事背景

```
1943 年，华盛顿。
美军轰炸机损失惨重，每次任务都有飞机有去无回。
军方收集了所有「返回飞机」的弹孔分布数据，制成图表。
指挥官找到你——数据分析师——只问一个问题：

"我们应该在飞机的哪里加装甲？"
```

---

## 三、游戏流程（四幕）

### 幕一：任务简报 `SceneBriefing`
- 打字机效果逐字显示背景故事
- 展示 B-17 轰炸机俯视图，标注真实弹孔分布（翼面、机身密集；发动机、驾驶舱稀少）
- 按钮：「开始调查」

### 幕二：审讯证人 `SceneInterrogation`
- 三位证人可自由选择顺序
- 点击证人 → 打开对话框 → 输入任何问题 → AI 动态回应
- 每位证人对话框有独立历史记录
- 底部按钮：「我已准备好，提交我的判断」

### 幕三：提交判断 `SceneVerdict`
- 展示飞机图，玩家点击「我认为应该加固这里」
- 可点击区域：左翼 / 右翼 / 机身前 / 机身中 / 机身后 / 左发动机 / 右发动机 / 驾驶舱
- **正确选择**：发动机区域 或 驾驶舱
- **错误选择**：任何弹孔密集区域
- 错误 → 动画展示后果（更多飞机被击落）→ 继续到揭示
- 正确 → 直接进入揭示，语气不同

### 幕四：真相揭示 `SceneRevelation`
- 动画：在飞机图上叠加「没有返回的飞机」的弹孔分布（与返回飞机完全相反）
- 文字揭示：你看到的，是幸存者的数据。你没看到的，才是真相。
- 引出概念：**幸存者偏差 → 选择偏误 → 样本偏差**
- 连接学术：「这就是计量经济学中「选择性偏误」的本质——你的样本不是随机的」
- 历史彩蛋：「这个洞见来自真实存在的人：Abraham Wald，1943年，他拯救了无数飞行员的生命」
- 结尾金句 + 重玩按钮

## 四、三位 AI 证人

| 证人 | 身份 | 性格 | 核心台词方向 |
|---|---|---|---|
| **Joe Kowalski** | 23岁，B-17 尾炮手，已完成 12 次任务 | 粗犷、直接、带点创伤 | 「我只知道我自己的飞机……那些没回来的兄弟，我不敢想」 |
| **Bill Henderson** | 35岁，地勤主任，见过所有返回飞机 | 务实、数字脑，不善理论 | 「奇怪，发动机几乎从来不在修理清单上——你会觉得它是个保护区一样」 |
| **Abraham Wald** | 38岁，统计研究小组数学家，匈牙利口音 | 精确、克制、每句话都值得细想 | 「我们看到的数据，只告诉我们飞机被击中后还能飞。我们没看到的数据，才告诉我们飞机被击中后会怎样」 |

**AI 行为准则**：
- 保持角色，不出戏
- 不直接说出「幸存者偏差」四个字
- 玩家问得越精准，回答越接近真相
- 玩家卡住时，给关键暗示而非答案

## 五、技术方案

```
技术栈：Vue 3 + Vite（纯前端，无后端）
样式：原生 CSS（1940s 军事档案风格）
AI：DeepSeek API（OpenAI 兼容格式）
飞机图：SVG（可点击区域）
状态管理：Vue 3 reactive（无需 Pinia）
```

### 文件结构
```
src/
├── App.vue                    # 根组件，管理当前幕次
├── api/
│   └── deepseek.js            # API 封装，换 key 只改 .env
├── components/
│   ├── SceneBriefing.vue      # 幕一：任务简报
│   ├── SceneInterrogation.vue # 幕二：审讯证人
│   ├── SceneVerdict.vue       # 幕三：提交判断
│   ├── SceneRevelation.vue    # 幕四：真相揭示
│   ├── PlaneMap.vue           # 可交互飞机 SVG
│   └── WitnessChat.vue        # 证人对话框（复用）
└── data/
    └── witnesses.js           # 三位证人的 system prompt
```

### DeepSeek 接入（核心代码）
```javascript
// src/api/deepseek.js
export async function askWitness(messages, systemPrompt) {
  const res = await fetch('https://api.deepseek.com/chat/completions', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${import.meta.env.VITE_DEEPSEEK_API_KEY}`
    },
    body: JSON.stringify({
      model: 'deepseek-chat',
      messages: [{ role: 'system', content: systemPrompt }, ...messages],
      temperature: 0.85,
      max_tokens: 200
    })
  })
  const data = await res.json()
  return data.choices[0].message.content
}
```

## 八、路演脚本（90秒版）

1. **痛点**（15秒）：「教材里讲选择偏误，一段文字，我看完觉得懂了——但考试的时候不会用。」
2. **演示**（60秒）：直接进游戏，走一遍审讯 → 答错 → 揭示
3. **价值**（15秒）：「这就是学习异化工厂——同样的知识，从「背下来」变成「亲身经历过」。Wald 的故事是真实的，这个顿悟也可以是真实的。」
